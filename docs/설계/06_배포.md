# 06. 배포 — Cloudflare Pages

> 배포처 확정: **Cloudflare Pages** (2026-07-03)
> 사이트 유형: 정적(Astro SSG) · 문의 폼: Web3Forms

---

## 1. 결정 배경

- 백엔드 없는 정적 회사 사이트 → 관리형 정적 호스팅이 유리(서버 운영 부담 0).
- Cloudflare Pages 선택 이유: **한국 속도 최상(서울 PoP)**, **대역폭 무제한 무료**, **무료 요금제 상업적 사용 허용**, DDoS·보안 기본 제공, DNS 통합.
- 자체 호스팅(Lightsail)은 정적 사이트엔 관리 비용 대비 실익이 낮아 제외. (Vercel은 무료 Hobby가 비상업용이라 제외.)

## 2. Cloudflare Pages 빌드 설정

| 항목 | 값 |
|------|-----|
| Framework preset | Astro |
| Build command | `npm run build` |
| Build output directory | `dist` |
| Node version | 20 이상 (필요 시 환경변수 `NODE_VERSION=20`) |

### 환경변수 (Settings > Environment variables)
| 이름 | 값 | 용도 |
|------|-----|------|
| `PUBLIC_WEB3FORMS_KEY` | (web3forms.com 발급 키) | 문의 폼 전송 |

## 3. 문의 폼 (Web3Forms)

- **키 발급**: https://web3forms.com 에서 수신용 이메일 입력 → access key 발급(문의는 그 이메일로 수신).
- 발급 키를 위 `PUBLIC_WEB3FORMS_KEY` 환경변수에 등록 → 재배포.
- 구현: [`src/pages/contact.astro`](../../src/pages/contact.astro) (`api.web3forms.com/submit` POST, botcheck 허니팟).
- 무료 티어: 월 250건. 초과/고급 기능 필요 시 유료.

## 4. 커스텀 도메인

- 대상 도메인: **nexto.co.kr** (www 여부 확정 필요 → 확정 후 [`src/config.yaml`](../../src/config.yaml) `site` URL 반영).
- Cloudflare Pages > Custom domains 에서 도메인 연결(네임서버를 Cloudflare로 이관하면 DNS·SSL 자동).
- 현재 `config.yaml` 값: `https://www.nexto.co.kr` (임시).

## 5. 배포 절차 (요약)

1. GitHub 저장소에 코드 push (현재 로컬은 git 미초기화 → 저장소 준비 필요).
2. Cloudflare Pages > Create project > Connect to Git → 저장소 선택.
3. 위 빌드 설정 + 환경변수 입력 → Deploy.
4. Web3Forms 키 등록 후 재배포 → 문의 폼 실제 전송 확인.
5. 커스텀 도메인 연결 → `config.yaml` site URL 최종 반영 → 재배포.

## 6. 사용자 액션 필요 항목 (체크리스트)

- [ ] Web3Forms access key 발급 → `PUBLIC_WEB3FORMS_KEY` 등록
- [ ] GitHub(또는 GitLab) 저장소 생성 + 코드 push
- [ ] Cloudflare Pages 프로젝트 생성·연결
- [ ] nexto.co.kr 도메인 연결(www 여부 결정)
