## 2. 폴더 구조 (AstroWind 기준)

```
Nexto-Site/
├── astro.config.ts            # Astro 통합(sitemap, mdx, icon, compress 등) + AstroWind 벤더 통합
├── src/
│   ├── config.yaml            # ★ 사이트명/SEO/i18n/블로그 설정 (브랜딩 1차 지점)
│   ├── navigation.ts          # ★ 헤더/푸터 메뉴 + 연락처 (브랜딩 2차 지점)
│   ├── components/
│   │   ├── Logo.astro         # ★ NEXTO 워드마크 (블루 "T")
│   │   ├── CustomStyles.astro # ★ 브랜드 컬러(--aw-color-*) + 폰트 변수
│   │   └── widgets/           # Hero, Features, Content, Steps, Stats, Contact, CTA ...
│   ├── assets/styles/tailwind.css  # Tailwind v4 진입 + Pretendard @import
│   ├── data/post/             # ★ 블로그 글(.md/.mdx) — 데이터 주도
│   ├── content.config.ts      # post 컬렉션 스키마
│   ├── layouts/               # PageLayout 등
│   └── pages/
│       ├── index.astro        # 홈
│       ├── about.astro        # 회사소개
│       ├── services.astro     # 제품·서비스
│       ├── contact.astro      # 문의
│       └── [...blog]/         # 블로그 목록/상세 (동적)
├── public/                    # 정적 자산
└── docs/                      # 설계/작업 문서 (보존)
```

> ⚠️ 제거한 데모: `pages/homes`, `pages/landing`, `pages/pricing.astro`, 영문 `terms/privacy`, AstroWind 데모 블로그 글.

## 3. 클론(확장) 전략 — 핵심

신규 사업(산업 자동화와 무관)은 **사이트 클론**으로 별도 운영한다. AstroWind 기준 클론 절차:

1. 레포 복제 → 새 디렉토리
2. **`src/config.yaml`** 교체 — 사이트명/도메인/SEO 메타데이터
3. **`src/navigation.ts`** 교체 — 메뉴/연락처
4. **`src/components/CustomStyles.astro`** 교체 — 브랜드 컬러(`--aw-color-primary` 등)·폰트
5. **`src/components/Logo.astro`** 교체 — 로고
6. **`src/data/post/*`**, `src/pages/*` 콘텐츠 교체, `public/` 자산 교체

→ 위 6개 지점만 손대면 코드 구조 변경 없이 새 사업 사이트 완성. (CLAUDE.md "상수화 / 변경 최소화 설계" 규칙과 연결)

## 4. i18n (한국어 우선)

- AstroWind는 `src/config.yaml`의 `i18n.language: ko`로 기본 언어를 한국어로 설정(현재 적용).
- v1은 한국어 단일 언어로 운영.
- **영어 확장(추후)**: AstroWind에 다국어 라우팅이 기본 내장되어 있지 않으므로, 영어가 필요해지면
  (a) `/en` 디렉토리에 영문 페이지를 복제하거나, (b) astro-i18next 등 i18n 통합을 추가한다.
  → P4 과제로 분리.
