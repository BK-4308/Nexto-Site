> 최초 작성: 2026-06-18

---

## 1. 사이트맵

```
/ (홈)
├── /about              회사소개
├── /services           제품·서비스 (목록)
│   └── /services/[slug] 제품·서비스 상세 (데이터 주도)
├── /blog               블로그/뉴스 (목록)
│   └── /blog/[slug]    글 상세 (마크다운)
└── /contact            문의/연락처/오시는 길
```

> i18n 확장 시: `/` = 한국어(기본, prefix 없음), `/en/...` = 영어. [04_아키텍처.md](04_아키텍처.md) §i18n 참조.

## 2. 글로벌 네비게이션

| 메뉴 | 경로 | 비고 |
|------|------|------|
| 회사소개 | /about | |
| 제품·서비스 | /services | 드롭다운(사업 영역)으로 확장 가능 |
| 소식 | /blog | |
| 문의 | /contact | 우측 강조(버튼형 CTA) |

- 로고(좌상단) 클릭 → 홈

## 3. 페이지별 콘텐츠 구성

### 3.1 홈 `/`
1. **Hero** — 슬로건("WE ARE ALWAYS NEXT TO YOU") + 한 줄 소개 + CTA(문의하기 / 서비스 보기)
2. **핵심 사업 영역** — 산업 자동화 / MES / 스마트 팩토리 / AI (카드 4종)
3. **회사 강점/숫자** — 신뢰 지표(연혁, 프로젝트 수 등 — 추후 실수치)
4. **대표 서비스 미리보기** — services 데이터 상위 3개
5. **최신 소식** — blog 최신 3개
6. **CTA 배너** — 문의 유도

### 3.2 회사소개 `/about`
- 회사 개요 / 비전·미션
- 핵심 역량 (자동화·MES·스마트팩토리·AI)
- 연혁(타임라인) — *추후 실제 데이터*
- 오시는 길 요약 → /contact 링크

### 3.3 제품·서비스 `/services`
- 사업 영역 그룹 + 서비스 카드 목록 (데이터 주도: `src/content/services`)
- 상세 `/services/[slug]`: 개요, 특징, 적용 분야, 문의 CTA

### 3.4 소식 `/blog`
- 카드형 목록(썸네일/제목/요약/날짜)
- 상세 `/blog/[slug]`: 마크다운 본문

### 3.5 문의 `/contact`
- 문의 폼(이름/연락처/이메일/문의유형/내용)
- 회사 연락처(주소, Tel, Fax, Email) — site config에서 출력
- 지도(오시는 길) — 임베드 placeholder

## 4. 콘텐츠 모델 (데이터 주도)

### service (제품·서비스)
| 필드 | 타입 | 설명 |
|------|------|------|
| title | string | 서비스명 |
| slug | string | URL |
| category | string | 사업 영역(예: 산업자동화/MES/스마트팩토리/AI) |
| summary | string | 한 줄 요약 |
| icon | string | 아이콘 키 |
| order | number | 정렬 |
| body | markdown | 상세 본문 |

### post (블로그/뉴스)
| 필드 | 타입 | 설명 |
|------|------|------|
| title | string | 제목 |
| slug | string | URL |
| date | date | 작성일 |
| summary | string | 요약 |
| tags | string[] | 태그 |
| cover | string? | 썸네일 |
| body | markdown | 본문 |
